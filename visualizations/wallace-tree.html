1. 数据通路的非均衡性

由于二进制乘法的移位特性，中间权重列 (如 2^7) 积累的位点远超边缘列 (如 2^0)。华莱士树通过动态分配 3:2 和 2:2 压缩器，在单一硬件时钟周期内最大化处理这种非均衡的数据路径，从而极大降低整体逻辑延迟。

2. 跨级无缝耦合设计

在“逻辑映射视图”中可以清晰观察到，当前列通过进位保留加法器 (CSA) 生成的 Cout 进位信号，不会在当前列发生堆积，而是直接接入下一阶段的高权位列 (2^{n+1})。这种设计彻底避免了长进位链带来的时序瓶颈。

3. 工业界主流架构选型：Wallace Tree vs Dadda Tree

在当前高性能计算芯片的乘法器 ALU 设计中，压缩阵列主要有两种技术流派：

  • Wallace Tree (本例演示)： 核心策略为“尽早压缩”。只要列高度足以例化压缩器，就立即进行规约。这种方式结构确定性高，但由于大量早期例化，会消耗较多逻辑门和横向布线面积。
  • Dadda Tree： 核心策略为“按需压缩”。将最大列高度严格控制在特定序列 (如 13, 9, 6, 4, 3, 2) 之内，只处理超高位点。这种方式能实现逻辑门总数的绝对最小化，是追求低功耗 (Low-Power) 和小面积设计的现代芯片更为倾向的选择。
